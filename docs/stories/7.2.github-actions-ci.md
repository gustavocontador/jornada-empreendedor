### Task 2: Job Lint Backend (AC: 2, 5)
- [x] Criar job `lint-backend`:
  ```yaml
  lint-backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'
      - run: pip install -r backend/requirements-dev.txt
      - run: ruff check backend/
      - run: black --check backend/
  ```
- [x] Adicionar cache de pip para performance

## Dev Notes

### Contexto do Projeto

O CI/CD é crítico para manter qualidade desde o dia 1. Este pipeline garante:
- **Zero broken builds:** Código quebrado nunca chega na main
- **Test coverage:** 80%+ obrigatório em backend e frontend
- **Lint enforcement:** Ruff (backend), ESLint (frontend)
- **Type safety:** mypy (backend), tsc (frontend)

### Arquitetura do Pipeline

```
CI Workflow (.github/workflows/ci.yml)
├── lint-backend (Python 3.12)
│   ├── Ruff (linting)
│   └── Black (formatting check)
├── lint-frontend (Node 20)
│   └── ESLint
├── test-backend (depends on lint-backend)
│   ├── PostgreSQL service container
│   ├── pytest + coverage
│   ├── mypy typecheck
│   └── Coverage threshold check (80%)
└── test-frontend (depends on lint-frontend)
    ├── Jest + coverage
    ├── tsc typecheck
    └── Coverage threshold check (80%)
```

### Implementation Strategy

**Abordagem:** Configuração completa de GitHub Actions CI Pipeline com quality gates rigorosos.

**Decisões técnicas:**
1. **Workflow structure:** Jobs paralelos (lint-backend, lint-frontend, test-backend, test-frontend) para maximizar performance
2. **Matrix builds:** Python 3.12 e Node 20 para garantir compatibilidade
3. **Service containers:** PostgreSQL 16 para testes de integração do backend
4. **Coverage enforcement:** Thresholds rigorosos (80%) com --cov-fail-under (pytest) e coverageThreshold (Jest)
5. **Type checking:** mypy (backend) e tsc (frontend) obrigatórios antes de merge
6. **Caching:** Automático via actions/setup-python@v5 e actions/setup-node@v4 para pip e npm
7. **Quality summary:** Job final que valida todos os outros jobs e falha se algum falhar

**Workspace dependency fix:**
- Mudou `"workspace:*"` → `"*"` nos package.json (backend, frontend)
- Protocolo `workspace:` é específico do pnpm/yarn, npm usa `"*"` para workspaces locais
